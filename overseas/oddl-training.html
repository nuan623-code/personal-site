是什么

Optimized Deferred Deep Linking（ODDL）

ODDL 通过将延迟深度链接的交付与归因处理彻底解耦，优化了用户从点击深度链接 → 安装/重装 App → 进入目标内容的完整旅程。 Deep link 通过更快的 session 响应通道立即下发，而不是等待归因响应完成后再交付。 系统始终基于距离安装/重装最近的点击来确定链接目标，确保路由准确性。

核心优势
⚡

更快的深度链接交付

从 6–8 秒降至 <1 秒，用户不再看到空白加载屏

🎯

更准确的路由

基于用户最近一次 Adjust 链接点击，路由到正确的 App 内容页

🔓

解除归因时序依赖

深度链接交付不再受 SAN API 等归因瀑布流时序影响

典型应用场景
🎬

媒体 / 流媒体

将用户直接跳转至指定节目、剧集或直播页面

🛍️

电商

安装后直接落地商品详情页，减少转化漏斗流失

🎟️

票务 / 活动

点击活动链接 → 安装 → 直接进入对应活动或场馆页

工作原理
❌ 旧方案（Before）
6–8 秒

Deep link 交付被归因流程阻塞。必须等待完整归因瀑布（包括 SAN API 响应）后才能下发链接，用户落地时只看到加载屏幕。

✅ ODDL（After）
< 1 秒

15 分钟内有 Adjust 链接点击时，Deep link 随 /session 响应立即返回，完全绕过归因流程，归因在后台异步继续执行。

⚡ 快速通道 Fast Path (ODDL)

用户在 15 分钟内点击过 Adjust 链接时触发。

  • Deep link 随 /session 响应立即返回
  • 归因在后台异步继续，不阻塞用户体验
  • 响应时间通常 < 1 秒
  • 基于距安装最近的点击确定目标内容
🔄 兜底路径 Legacy Fallback

15 分钟窗口外，或无符合条件点击时触发。

  • 回退至原有归因依赖路径
  • 等待完整归因响应后再交付 deep link
  • 行为与旧方案一致，无额外风险
已知限制
⏱️

15 分钟点击窗口

仅当用户在安装/重装前 15 分钟内点击过 Adjust 链接时，才会触发 Fast Path。超出此窗口则自动回退至 Legacy 路径。

⚠️

自归因网络（SAN）不受 Fast Path 覆盖

Google、Meta、Apple Search Ads 等 SAN 通过独立的 API / Pipeline 进行归因，Adjust 无法实时获取其点击数据。等待 SAN API 响应会与 ODDL 的设计目标相悖，因此 SAN 流量仍走 Legacy 归因路径。

🔧

需开启概率匹配（Probabilistic Modeling）

当设备标识符不可用时，系统需依赖概率匹配来识别点击与安装的关联关系。必须在 App 的归因设置中提前启用此选项，否则 ODDL 无法正常工作。

如何开启（客户自助）
1

升级 Adjust SDK

需升级至以下版本或更高：

iOS 5.4.4+ Android 5.4.3+ Unity 5.4.3+ React Native 5.4.3+ Flutter 5.4.3+ Cordova 5.4.3+
✅ 仅需客户端升级，Adjust 后端无需工程改动
2

开启概率匹配（Probabilistic Matching）

在 App 的归因设置中启用概率匹配。设备标识符不可用时，系统将依赖此能力识别点击。

3

使用现有 Adjust 深度链接

无需修改或新建链接，沿用客户现有的 Adjust Deep Link 即可。

4

开启 ODDL 开关

进入 AppView → Advanced Settings → Optimized Deferred Deep Linking，将开关设为 ON（默认关闭）。

🎉 开启即生效，无需联系 Adjust 工程师
测试流程

📲 安装测试（Install Test）

  1. 安装 App
  2. 从日志中获取设备的 ADID
  3. 卸载 App
  4. 在 Testing Console 中确认设备状态
  5. 在 Console 中 Forget（遗忘）该设备
  6. 点击 Adjust 深度链接
  7. 通过本地构建包重新安装 App
  8. 打开 App，确认目标深度链接内容正常显示

🔄 重装测试（Reinstall Test）

  1. 卸载 App
  2. 等待 30 分钟
  3. 点击 Adjust 深度链接
  4. 通过本地构建包重新安装 App
  5. 打开 App，确认目标深度链接内容正常显示
💡 注意：重装测试中等待 30 分钟是为了确保系统将其识别为重装（而非新安装），与 ODDL 15 分钟窗口不冲突——点击发生在重装之前。
